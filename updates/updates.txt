DTWAIN-Java Version changes

Version 1.9.4

- DTWAIN library (DLL's and text resource files) version 5.8.0 is required for this version.

- With the introduction of DTWAIN 5.8.0, the TWAIN Sources that have had issues such as 
  acquiring images, non-responsive UI, etc. has been addressed.  See the issue link:
  https://github.com/dynarithmic/twain_library/issues/188

- Updated dtwainjni.info file.  You must use this version of the info file for version 1.9.4 (or above).

- Allow TwainSource.acquire() to start a "show UI-Only" dialog if user-interface option 
  has showUIOnly() as true.

- Added com.dynarithmic.dtwain.demos.ShowUIOnlyDemo.java demo program.

- Added TwainSource functions getSaveFileName() and setSaveFileName() to retrieve
  the name of the image file that will be saved, and also changing the name of the
  image file that will be saved.  These functions should be utilized while the device is 
  acquiring images.  See the TwainCallbck handlers for onFileNameChanging() and 
  onFileNameChanged(), as this is where getSaveFileName() and setSaveFileName() will have
  any effect.

- Added TwainCallback handlers for onFileNameChanging() and onFileNameChanged() to allow the
  retrieval and setting of the file name that will be saved (see above for getSaveFileName()
  and setSaveFileName()).

- Added centerCurrentMonitor() option to the com.dynarithmic.twain.highlevel.TwainSourceDialog
  class.  This allows the "Select Source" TWAIN dialog to be centered on the current monitor where
  the main application window is located.  

- TwainSource.writePDFTextElement() will now only work when the current page is being acquired.  
  As a result, the DTwainConstants.TextPageDisplayOptions enumeration has been removed.  For
  an example, see the com.dynarithmic.dtwain.demos.PDFAddTextToPageDemo.java program.

- com.dynarithmic.twain.highlevel.PDFTextElement now allows text rotation, x/y scaling, skewing, plus
  transformation order for these operations.


Version 1.9.3

- Add getFileSavePageCount() function to the TwainSource class.

- Update additional error constants to DTwainConstants.java.

- Update TwainCallback.java to add new callbacks that were added to 
  DTWAIN API 5.7.1 and above.


Version 1.9.2

- Fix issue when setting some PDF page size and scale options.


Version 1.9.1

- Fix issue when enabling feeder on devices with both a feeder and flatbed.

- Add missing DTWAIN constants.


Version 1.9 

- Update JNI DLL's to be compatible with DTWAIN version 5.7.0.  

- Update dtwain-java jar file to dtwain-java-1.9.jar.  

- Add JPEG-XR file format when acquiring to files and update com.dtwain.demos.fulldemo 
  with acquiring to files using the JPEG-XR format.

- Add AES 256-bit encryption of PDF files.  
  See com.dynarithmic.twain.highlevel.acquirecharacteristics.PDFOptions 
  with the new function being enableAES256Encryption().


Version 1.8.1 

- Update JNI DLL's to be compatible with DTWAIN version 5.6.8.  

- Update dtwain-java jar file to dtwain-java-1.8.jar.  

- In the Java library code, add AES 128-bit encryption of PDF files.  See 
  com.dynarithmic.twain.highlevel.acquirecharacteristics.PDFOptions with the new function
  being enableAESEncryption().


Version 1.8 (JNI code changes only)

- Update JNI DLL's to be compatible with DTWAIN version 5.6.7.  Note that you must use the
  text resources for DTWAIN version 5.6.7, as older versions of the text resources will not
  work for JNI version 1.8 DLL's. 


Version 1.7.1 

- Update JNI DLL's to be compatible with DTWAIN 5.6.3.  

- In com.dtwain.demos.fulldemo, ensure TWAIN is shut-down properly when closing the demo program
  by clicking the "X" (close) button on the GUI.

- Update dtwain-java jar file to dtwain-java-1.7.jar.  


Version 1.7 (JNI code changes only)

Update JNI DLL's to be compatible with DTWAIN 5.6.2.  No java manifest files were updated.


Version 1.6

- Ensure proper handling of ANSI strings when calling capability functions that require strings.

- Fix issue with capabilityInterface.setCapValues() ignoring the capability and data type settings.


Version 1.5

- Allow application to prioritize and enable/disable scanning options and capabilities in the 
  OptionsApplyer class.

- Add AutoScanningOptions() to OptionsApplyer.

- Add missing flip-rotation and other options to OptionApplyer.

- Set certain AcquireCharacteristics values to proper TWAIN default values.


Version 1.4

- Add ExtendedImageInfo.java to allow retrieval of extended image information such as barcode text
  and positioning, imprinter text, line speckle removal, etc.  See the TWEI_... constants and
  definitions in the TWAIN 2.x specification for more information.

- Added ExtendedImageInfoDemo.java to demonstrate usage of the ExtendedImageInfo class.

- Updated BarcodeDemo.java to output the information on the barcodes that have been found on the 
  acquired page.  Note that your device must support barcode retrieval, and support extended image
  information that will report barcode information (see update above).

- The dtwainjni.info file has been updated.  You must use this updated dtwainjni.info file with 
  version 1.4 of DTWAIN-Java, as any earlier versions of dtwainjni.info will not work (an 
  exception will be thrown back to Java if the file is not valid).  

- You must use the DTWAIN DLL's (dtwain32u.dll, dtwain64u.dll, etc.) that are of version 5.5.4 or higher.  
  Earlier versions of DTWAIN will not work correctly with version 1.4 of DTWAIN_Java.


Version 1.3

- Fix issue with JNI layer not maintaining global class object for logger and callback Java classes.

- C++/JNI layer no longer has the requirement for DTWAIN import libraries to be used when building
  the JNI DLL's.
